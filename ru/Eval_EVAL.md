# Выполнение программного кода (EVAL)

Оператор выполнения программного кода создает [действие](/ru/Actions/.md), которое выполняет заданный программный код на языке **lsFusion**. Программный код задается как некоторое [свойство](/ru/Properties/.md), значение которого должно принадлежать строковому классу.

Код, который передается действию для выполнения, должен быть последовательностью инструкций. Предполагается, что одной из этих инструкций будет создание действия с именем `run` (именно это действие и будет выполняться).

### Выполнение действия[​](#выполнение-действия "Прямая ссылка на этот заголовок")

В этом операторе также существует возможность выполнить просто действие (а не набор инструкций), в этом случае код должен быть последовательностью [операторов-действий](/ru/Action_operators_paradigm/.md) и объявлений локальных свойств. Для обращения к параметрам выполняемого действия можно использовать спецсимвол `$` и номер параметра (начиная с `1`).

### Ограничения на выполняемый код[​](#ограничения-на-выполняемый-код "Прямая ссылка на этот заголовок")

В текущей реализации платформы выполняемый код не должен содержать инструкций, которые изменяют уже существующие объекты в системе. Например нельзя добавлять [события](/ru/Events/.md), [первичные](/ru/Data_properties_DATA/.md) и материализованные свойства, [расширять](/ru/Extensions/.md) существующие объекты и т.п.

### Язык[​](#язык "Прямая ссылка на этот заголовок")

Для объявления действия, выполняющего программный код, используется [оператор `EVAL`](/ru/EVAL_operator/.md).

### Примеры[​](#примеры "Прямая ссылка на этот заголовок")

```
// исходный код, состоящий из одной инструкции, задается строковой константой
addProperty { EVAL ACTION 'MESSAGE \'Hello World\''; }  

// свойство, в которое пользователь может ввести исходный код
code 'Исходные код' = DATA BPSTRING[2000] ();

// действие, выполняющее этот код
execute 'Выполнить код'  { EVAL code(); } 
```
