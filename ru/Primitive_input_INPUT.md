# Ввод значения (INPUT)

Оператор *ввода значения* создаёт [действие](/ru/Actions/.md), которое запрашивает у пользователя [ввод значения](/ru/Value_input/.md). Запрашиваемое значение может быть как объектом [встроенного класса](/ru/Built-in_classes/.md) (например, строкой, числом, датой или файлом), так и объектом пользовательского класса. Пользователь при желании может [отменить ввод](/ru/Value_input/.md#result), например, нажав на клавиатуре клавишу `Esc`.

Ввод значения встроенного класса выполняется в редакторе того свойства формы, изменение которого обрабатывается, поэтому для большинства встроенных классов оператор нужно вызывать из обработки [события изменения](/ru/Form_events/.md#property) свойства. Значения файловых классов и цвета вводятся через отдельный диалог, поэтому их можно запрашивать и из других обработчиков событий формы, например по нажатию кнопки.

### Запрашиваемое значение[​](#value "Прямая ссылка на этот заголовок")

Если запрашивается объект встроенного класса, платформа показывает пользователю [элемент ввода](/ru/Value_input/.md#execution), соответствующий этому классу, и пользователь вводит его напрямую.

Если запрашивается объект пользовательского класса, пользователь не вводит его напрямую, а выбирает из [формы выбора](/ru/Interactive_view/.md#edtClass) объектов класса.

### Предлагаемые значения[​](#list "Прямая ссылка на этот заголовок")

Значения, предлагаемые пользователю для выбора, можно ограничить заданным набором, а этот набор дополнительно сузить условием. Для встроенного класса такой набор показывается пользователю как список подсказок при вводе; для пользовательского класса он ограничивает объекты, доступные для выбора.

Кроме того, если ввод используется для [изменения](/ru/Value_input/.md#initial) свойства, предлагаемые значения по умолчанию дополнительно ограничиваются теми, выбор которых не нарушит существующих в системе [ограничений](/ru/Constraints/.md) — так же, как в [диалоговой форме](/ru/In_an_interactive_view_SHOW_DIALOG/.md#dialog) открытия формы. При необходимости это ограничение можно отключить.

### Общие возможности ввода[​](#общие-возможности-ввода "Прямая ссылка на этот заголовок")

Также как и в остальных операторах ввода значения, в этом операторе можно:

* задавать [начальные значения](/ru/Value_input/.md#initial) объектов
* задавать [основное и альтернативное](/ru/Value_input/.md#result) действия. Первое вызовется, если ввод был успешно завершён, второе - в обратном случае (если ввод был отменён).
* [осуществлять изменение](/ru/Value_input/.md#initial) заданного свойства

### Язык[​](#язык "Прямая ссылка на этот заголовок")

Синтаксис оператора ввода значения описывается [оператором `INPUT`](/ru/INPUT_operator/.md).

### Примеры[​](#примеры "Прямая ссылка на этот заголовок")

```
changeCustomer (Order o)  {
    INPUT s = STRING[100] DO {
        customer(o) <- s;
        IF s THEN
            MESSAGE 'Customer changed to ' + s;
        ELSE
            MESSAGE 'Customer dropped';
    }
}

FORM order
    OBJECTS o = Order
    PROPERTIES(o) customer ON CHANGE changeCustomer(o)
;

testFile  {
    INPUT f = FILE DO { // запрашиваем диалог по выбору файла
        open(f); // открываем выбранный файл
    }
}

sku = DATA Sku (OrderDetail);

// выбор объекта пользовательского класса: предлагаем sku по их наименованию и оставляем только те, что в наличии
changeSku (OrderDetail d)  {
    INPUT s = sku(d) CHANGE LIST name(s) WHERE inStock(s);
}
```
