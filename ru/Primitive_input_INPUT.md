# Ввод примитива (INPUT)

Оператор *ввода примитива* создает [действие](/ru/Actions/.md), которое запрашивает у клиента [ввод значения](/ru/Value_input/.md) [встроенного класса](/ru/Built-in_classes/.md). Пользователь при желании может [отменить ввод](/ru/Value_input/.md#result), например, нажав на клавиатуре клавишу `Esc`.

Также как и в остальных операторах ввода значения, в этом операторе можно:

* задавать [начальные значения](/ru/Value_input/.md) объектов
* задавать [основное и альтернативное](/ru/Value_input/.md#result) действия. Первое вызовется, если ввод был успешно завершен, второе - в обратном случае (если ввод был отменен).
* [осуществлять изменение](/ru/Value_input/.md) заданного свойства

Этот оператор можно использовать только в обработке [событий изменения](/ru/Form_events/.md#property) свойства на форме.

### Язык[​](#язык "Прямая ссылка на этот заголовок")

Синтаксис оператора ввода значений описывается [оператором `INPUT`](/ru/INPUT_operator/.md).

### Примеры[​](#примеры "Прямая ссылка на этот заголовок")

```
changeCustomer (Order o)  {
    INPUT s = STRING[100] DO {
        customer(o) <- s;
        IF s THEN
            MESSAGE 'Customer changed to ' + s;
        ELSE
            MESSAGE 'Customer dropped';
    }
}

FORM order
    OBJECTS o = Order
    PROPERTIES(o) customer ON CHANGE changeCustomer(o)
;

testFile  {
    INPUT f = FILE DO { // запрашиваем диалог по выбору файла
        open(f); // открываем выбранный файл
    }
}
```
