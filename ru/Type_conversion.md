# Преобразование типа

Оператор *преобразования типа* создает [свойство](/ru/Properties/.md), которое преобразует объект одного [встроенного класса](/ru/Built-in_classes/.md) к объекту другого встроенного класса. Результат принадлежит целевому встроенному классу. Если преобразование типа невозможно, значением свойства будет `NULL`.

### Поддерживаемые преобразования[​](#поддерживаемые-преобразования "Прямая ссылка на этот заголовок")

Какие преобразования имеют смысл, зависит от [семейств](/ru/Built-in_classes/.md#inheritance) встроенных классов, к которым принадлежат исходный и целевой классы:

* В пределах семейства чисел значение сохраняется настолько, насколько это допускает целевой класс; если значение не помещается в целевой класс, результатом является `NULL`.
* Между числами и строками значение преобразуется между числовой и текстовой формами; если исходную строку нельзя прочитать как число целевого класса, результатом является `NULL`.
* Строки также преобразуются в классы `JSON`, `JSONTEXT`, `XML` и `HTML` и обратно.

### Строковые и файловые типы[​](#строковые-и-файловые-типы "Прямая ссылка на этот заголовок")

Строковые типы можно преобразовывать в файловые human-readable типы (`CSVFILE`, `XMLFILE`, `JSONFILE`, `HTMLFILE` и т.д.), и наоборот - файловые human-readable типы можно преобразовывать в строковые типы. Файл конкретного типа также можно преобразовать в файл динамического типа (`FILE`, `NAMEDFILE`) и обратно с сохранением содержимого файла; файл динамического типа дополнительно несёт расширение, а `NAMEDFILE` — ещё и имя.

к сведению

Преобразовывать файлы динамического типа (`FILE`) к строкам и наоборот в текущей реализации запрещено, но, при необходимости, это можно сделать через промежуточный human-readable тип - то есть преобразовать сначала, к примеру, к `CSVFILE`, а уже потом к `FILE` (при этом у результирующего файла [будет расширение](/ru/Built-in_classes/.md#extension) csv)

### Неявные преобразования[​](#неявные-преобразования "Прямая ссылка на этот заголовок")

Помимо явного оператора преобразования, платформа в ряде случаев преобразует значение к другому классу неявно, без написанного разработчиком преобразования:

* Когда объединяются значения разных, но совместимых классов — при сложении или вычитании (см. [арифметические операторы](/ru/Arithmetic_operators_plus_minus_etc/.md)), в [операторах выбора](/ru/Selection_CASE_IF_MULTI_OVERRIDE_EXCLUSIVE/.md) или в [операторах экстремума](/ru/Extremum_MAX_MIN/.md) — каждое значение преобразуется к их [общему предку](/ru/Built-in_classes/.md#commonparentclass), который становится классом результата; в частности, так расширяются числовые классы (`INTEGER`, объединённый с `NUMERIC`, становится этим `NUMERIC`).
* При [объединении строк](/ru/String_operators_plus_CONCAT_SUBSTRING/.md) операнд, не являющийся строкой, преобразуется к строке.
* Когда значение записывается в свойство другого, но совместимого класса, оно преобразуется к этому классу.
* Файл конкретного типа преобразуется к файлу динамического типа (`FILE`, `NAMEDFILE`) неявно — например, при [импорте данных](/ru/Data_import_IMPORT/.md) без указания формата или при [взаимодействии с внешней системой](/ru/Access_to_an_external_system_EXTERNAL/.md); результирующее расширение определяется так же, как при явном преобразовании.

### Язык[​](#язык "Прямая ссылка на этот заголовок")

Для реализации преобразования используется [оператор преобразования типа.](/ru/Type_conversion_operator/.md)

### Примеры[​](#примеры "Прямая ссылка на этот заголовок")

```
itemCount = DATA INTEGER (Store);
itemCountToString(s) = BPSTRING[10](itemCount(s));

barcode = DATA STRING[15] (Item);
longBarcode(Item i) = LONG(barcode(i));
```
