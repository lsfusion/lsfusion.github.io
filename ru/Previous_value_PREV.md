# Предыдущее значение (PREV)

Оператор *предыдущего значения* создает [свойство](/ru/Properties/.md), которое возвращает значение заданного свойства на начало [сессии изменений](/ru/Change_sessions/.md) (то есть текущее значение в базе данных без учета изменений, сделанных в сессии). Вместе с производными от него [операторами изменений](/ru/Change_operators_SET_CHANGED_etc/.md) этот оператор обеспечивает отслеживание изменений в сессии: он дает доступ к состоянию "до", с которым сравнивается текущая сессия.

к сведению

Этот оператор всегда использует область видимости начала сессии. О том, как это взаимодействует с [обработкой событий](/ru/Events/.md#change), см. канонический раздел про событийный режим.

### Язык[​](#язык "Прямая ссылка на этот заголовок")

Для объявления свойства, возвращающего предыдущее значение, используется [оператор `PREV`](/ru/PREV_operator/.md).

### Примеры[​](#примеры "Прямая ссылка на этот заголовок")

```
CLASS Order;
sum = DATA NUMERIC[10,2] (Order);

// Самый частый случай — чтение значения "до" внутри обработчика события, реагирующего
// на изменение. PREV(sum(o)) возвращает значение на начало сессии — значение из базы,
// игнорирующее изменения в текущей сессии. Предикаты изменения (CHANGED, SET, DROPPED,
// ...) являются производными от PREV — это булевы сравнения с текущим значением, а сам
// PREV даёт доступ к фактическому предыдущему значению.
WHEN CHANGED(sum(Order o)) DO
    MESSAGE 'Сумма изменена: ' + PREV(sum(o)) + ' → ' + sum(o);
```
