# Обращение к внутренней системе (INTERNAL, FORMULA)

Под *внутренней системой* подразумеваются компоненты развёртывания самой lsFusion-системы — JVM сервера приложений, веб-клиент пользователя и собственная БД платформы. Для обращения из lsFusion-кода к этим компонентам платформа предоставляет два разных оператора: [`INTERNAL`](/ru/Internal_call_INTERNAL/.md) — выполнение Java/JavaScript/SQL-кода как [действия](/ru/Actions/.md), и [`FORMULA`](/ru/Custom_formula_FORMULA/.md) — оборачивание SQL-выражения в [свойство](/ru/Properties/.md).

### Java-взаимодействие[​](#javato "Прямая ссылка на этот заголовок")

Для реализации такого типа взаимодействия, в платформе используется оператор [внутреннего вызова (`INTERNAL`)](/ru/Internal_call_INTERNAL/.md), который позволяет вызвать код на языке Java внутри JVM lsFusion-сервера. То, что доступно с другой стороны такого вызова — из самого Java-кода — описано в статье [обращение из внутренней системы](/ru/Access_from_an_internal_system/.md).

### SQL-взаимодействие[​](#sql-взаимодействие "Прямая ссылка на этот заголовок")

Для реализации такого типа взаимодействия, в платформе используется оператор [пользовательской формулы (`FORMULA`)](/ru/Custom_formula_FORMULA/.md), который позволяет обращаться к объектам / синтаксическим конструкциям SQL-сервера, используемого разрабатываемой lsFusion-системой. Для произвольных SQL-команд, выполняемых как действие в текущей [сессии изменений](/ru/Change_sessions/.md), используется форма [`INTERNAL DB`](/ru/Internal_call_INTERNAL/.md#db).
