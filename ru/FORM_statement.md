# Инструкция FORM

Инструкция `FORM` - создание [формы](/ru/Forms/.md).

## Синтаксис[​](#синтаксис "Прямая ссылка на этот заголовок")

```
FORM name [caption] formOptions
    formBlock1
    ...
    formBlockN
;
```

После указания имени и заголовка формы в произвольном порядке указываются опции формы `formOptions`:

```
imageSetting
LOCALASYNC
```

После опций формы описываются в произвольном порядке блоки формы `formBlock1 ... formBlockN`:

```
OBJECTS ... 
TREE ...
PROPERTIES ...
FILTERS ...
[EXTEND] FILTERGROUP ...
USERFILTERS ...
ORDERS ...
PIVOT ...
EVENTS ...
REPORT propertyExpression
FORMEXTID extID
EDIT className OBJECT objectName
LIST className OBJECT objectName 
```

## Описание[​](#описание "Прямая ссылка на этот заголовок")

Инструкция `FORM` объявляет новую форму и добавляет ее в текущий [модуль](/ru/Modules/.md). Кроме того эта инструкция позволяет описать [структуру формы](/ru/Form_structure/.md), а также ее [статичное](/ru/Static_view/.md) и частично [интерактивное](/ru/Interactive_view/.md) (за исключением [дизайна формы](/ru/Form_design/.md)) представление. В начале инструкции задаются имя и заголовки, затем указываются опции формы, а затем объявление содержит произвольное количество блоков, описывающих структуру формы. Их можно использовать в любом порядке, при условии, что каждый блок объявляется позже блоков, элементы которых он использует. Каждый блок может использоваться произвольное количество раз.

## Параметры[​](#параметры "Прямая ссылка на этот заголовок")

* `name`

  Имя формы. [Простой идентификатор](/ru/IDs/.md#id). Имя должно быть уникально в пределах текущего [пространства имен](/ru/Naming/.md#namespace).

* `caption`

  Заголовок формы. [Строковый литерал](/ru/IDs/.md#strliteral). Если заголовок не задан, то заголовком формы будет являться ее имя.

### Опции формы (`formOptions`)[​](#опции-формы-formoptions "Прямая ссылка на этот заголовок")

* `imageSetting`

  Настройка иконки формы. Эта опция позволяет настроить отображение иконки вручную. Может иметь один из двух видов:

  * `IMAGE [imageLiteral]`

    [Указание иконки](/ru/Icons/.md#manual) формы. Если `imageLiteral` не указывается, то происходит переключение в режим [автоматической установки](/ru/Icons/.md#auto) иконки.

    * `imageLiteral`

      Строковый литерал, значение которого определяет иконку.

  * `NOIMAGE`

    Ключевое слово, указывающее на то, что иконка формы должна отсутствовать.

* `LOCALASYNC`

  Ключевое слово, указывающее на то, что обработки [локальных событий](/ru/Events/.md#type) будут выполняться после отображения изменений на форме.

### Блоки формы (`formBlock1 ... formBlockN`)[​](#blocks "Прямая ссылка на этот заголовок")

* `OBJECTS ...`

  Добавляет объекты на форму. [Синтаксис блока объектов](/ru/Object_blocks/.md).

* `TREE ...`

  Добавляет дерево объектов на форму. [Синтаксис блока дерева объектов](/ru/Object_blocks/.md#tree).

* `PROPERTIES ...`

  Добавляет [свойства](/ru/Properties/.md) и [действия](/ru/Actions/.md) на форму. [Синтаксис блока свойств и действий](/ru/Properties_and_actions_block/.md).

* `FILTERS ...`

  Добавляет фиксированные фильтры на форму. [Синтаксис блока фиксированных фильтров](/ru/Filters_and_sortings_block/.md#fixedfilters).

* `[EXTEND] FILTERGROUP ...`

  Добавляет группу фильтров на форму либо расширяет существующую. [Синтаксис блока группы фильтров](/ru/Filters_and_sortings_block/.md#filtergroup).

* `USERFILTERS ...`

  **deprecated since version 7**, используйте `FILTERS` с опцией `USER`. Добавляет [пользовательские фильтры](/ru/Interactive_view/.md#userfilters) на форму. [Синтаксис блока пользовательских фильтров](/ru/Filters_and_sortings_block/.md#userfilters).

* `ORDERS ...`

  Добавляет сортировки на форму. [Синтаксис блока сортировок](/ru/Filters_and_sortings_block/.md#sort).

* `PIVOT ...`

  Устанавливает начальные настройки [вида представления *сводная таблица*](/ru/Interactive_view/.md#property). [Синтаксис блока сводных таблиц](/ru/Pivot_block/.md).

* `EVENTS ...`

  Устанавливает действия, выполняемые при определенных событиях. [Синтаксис блока событий](/ru/Event_block/.md).

* `FORMEXTID extID`

  Указание имени, которое будет использоваться для [экспорта / импорта](/ru/Structured_view/.md#extid) этой формы. Используется только в [структурированном](/ru/Structured_view/.md) представлении.

  * `extId`

    Строковый литерал.

* `REPORT propertyExpression`

  Указание свойства, значение которого будет использовано в качестве имени файла [отчета](/ru/Print_view/.md) для [пустой](/ru/Static_view/.md#empty) группы. В качестве параметров можно использовать имена уже объявленных объектов на форме. Предполагается, что значения этих объектов будут [переданы](/ru/Open_form/.md) при открытии формы [в печатном представлении](/ru/In_a_print_view_PRINT/.md) (если этого не будет сделано, они будут считаться равными `NULL`).

  * `propertyExpression`

    [Выражение](/ru/Expression/.md).

* `EDIT сlassName OBJECT objectName`

  Устанавливает текущую форму в качестве формы [редактирования](/ru/Interactive_view/.md#edtClass) объектов указанного класса.

  * `className`

    Имя [пользовательского](/ru/User_classes/.md) класса, при редактировании объектов которого будет вызываться описываемая форма. Задается [составным идентификатором](/ru/IDs/.md#cid).

  * `objectName`

    Имя объекта формы, который будет использоваться для редактирования. Задается [простым идентификатором](/ru/IDs/.md#id).

* `LIST сlassName OBJECT objectName`

  Устанавливает текущую форму в качестве формы [выбора](/ru/Interactive_view/.md#edtClass) объекта указанного класса.

  * `className`

    Имя [пользовательского](/ru/User_classes/.md) класса, для выбора объекта которого будет вызываться описываемая форма. Задается составным идентификатором.

  * `objectName`

    Имя объекта формы, текущее значение которого будет использоваться в качестве выбираемого объекта. Задается простым идентификатором.

## Примеры[​](#примеры "Прямая ссылка на этот заголовок")

```
CLASS Document;

// объявляем форму Документы
FORM documents 'Документы'
    // Добавляем один объект класса Document, по этому имени объект будет доступен в операторах
    // DESIGN, SHOW, EXPORT, DIALOG и др.
    OBJECTS d = Document 

    // ... добавляем свойства и фильтры на форму

    // помечаем что данная форма должна использоваться, когда необходимо выбрать документ, 
    // при этом объект d должен использоваться как возвращаемое значение
    LIST Document OBJECT d 
;

CLASS Item;

// объявляем форму Товар
FORM item 'Товар'
    // добавляем объект класса Item и помечаем, что он должен отображаться в панель
    // (то есть видимо только одно значение)
    OBJECTS i = Item PANEL 

    // ... добавляем свойства и фильтры на форму

    // помечаем что данная форма должна использоваться, когда необходимо добавлять или редактировать товар
    EDIT Item OBJECT i 
;

// объявляем форму со списком Товаров
FORM items 'Товары'
    OBJECTS i = Item

    // ... добавляем свойства и фильтры на форму

    // добавляем кнопки, которые будут создавать и редактировать товар, используя форму item
    PROPERTIES(i) NEWSESSION NEW, EDIT 
;

CLASS Invoice;
CLASS InvoiceDetail;

// объявляем форму печати инвойса
FORM printInvoice
    OBJECTS i = Invoice // добавляем объект класса инвойс, для которого будет идти печать

    // ... добавляем свойства и фильтры на форму
;

// разбиваем определение формы на две инструкции (вторую инструкцию можно перенести в другой модуль)
EXTEND FORM printInvoice
    // добавляем строки инвойса, каждая из которых будет использоваться в отчете в качестве детализации
    OBJECTS d = InvoiceDetail 

    // ... добавляем свойства и фильтры на форму
;
// объявляем действие , при выполнении которого будет открываться форма печати инвойса
print (Invoice invoice)  { PRINT printInvoice OBJECTS i = invoice; } 
```
